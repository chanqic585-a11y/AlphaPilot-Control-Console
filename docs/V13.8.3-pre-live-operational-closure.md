# AlphaPilot V13.8.3 - Pre-Live Operational Closure Pack

V13.8.3 closes the local pre-live preparation loop without enabling testnet,
live trading, API keys, private exchange requests, or order creation.

## What Changed

- Added persistent local pre-live rehearsal records.
- Added a closure report for success-path and rejection-path rehearsal evidence.
- Added recent rehearsal records to the desktop control console.
- Added a local operational runbook for pre-live readiness review.
- Added `POST /api/pre-live-order-lifecycle/rehearse`.

## Local Rehearsal Records

The rehearsal endpoint runs the existing local lifecycle simulator and saves the
result into local console state as an audit record.

Saved fields include:

- `rehearsalId`
- `strategyId`
- `symbol`
- `notional`
- `riskR`
- `manualDecision`
- `riskPassed`
- `finalState`
- `lifecyclePath`
- `riskNotes`
- safety flags showing no exchange order, no API key, and no private exchange
  connection

## Closure Report

The closure report checks:

- whether any local rehearsal has been saved
- whether a passed local path exists
- whether a rejected local path exists
- whether manual confirmation remains required
- whether credential-vault work is still future-required
- whether exchange execution remains disabled

## Safety Boundary

V13.8.3 does not add:

- API key input
- API key storage
- Trade API
- Withdraw API
- real account reads
- real position reads
- real or testnet order creation
- exchange Dry-run
- live trading
- automatic trading

The control console remains a local research and operations dashboard only.
