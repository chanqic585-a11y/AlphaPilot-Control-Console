# AlphaPilot V13.27.3 - Short-Cycle Workflow Candidate Pack

V13.27.3 makes the main strategy lifecycle operable one candidate at a time or
in controlled batches:

```text
Strategy -> Formal Backtest -> Local Forward -> OKX Demo -> Live Candidate
```

## Candidate Pack

The Quant Engine registers ten research-only candidates:

- Five strategies with a 5-minute signal and 5-minute execution horizon.
- Five strategies with a 15-minute signal and 5-minute execution horizon.
- Every strategy has `targetR >= 2`, a frozen signal definition, and public
  market data only before Demo.

Registration does not run a backtest and does not create a StrategyCandidate,
DemoRelease, LiveCandidatePackage, or LiveRelease.

## Formal Backtest Contract

Formal promotion uses a historical point-in-time dynamic Top50 universe. Each
historical decision can only use instruments and liquidity information that
were available at that time. Today's static Top50 is never projected backward.

Single-symbol runs are allowed only for smoke/debug. They cannot count as
formal promotion evidence. Formal evaluation retains fees, slippage, funding,
delay, purged walk-forward, locked OOS, regime splits, and unseen-symbol
checks.

Selected backtests run through one serial worker. This protects CPU, memory,
disk IO, checkpoints, and the shared registry from competing heavy jobs.

## Local Forward Contract

Each formally passed strategy receives its own immutable public-only forward
release, virtual session, state checkpoint, and outcome ledger. The Local page
can run one strategy, selected strategies, or all eligible strategies. Batch
cycles are serial and only consume completed public candles.

Local forward does not use an API key, does not connect to a private endpoint,
and does not create an exchange order. Re-reading the same completed candle
does not manufacture an independent closed sample.

## Demo Controls

Demo cards expose one-card, selected, and all-eligible controls. A batch takes a
snapshot of each card's current legal next action and executes only that one
step. It cannot cross multiple gates in one click and it never invokes the
controlled-override action.

OKX Demo execution still requires an immutable Demo Release, process-only Demo
credentials, read-only preflight, the order and automation gates, RiskProfile,
idempotency, and reconciliation. Live and Withdraw remain locked.

## Safety Boundary

- No raw API key storage.
- No Withdraw API or key-management endpoint.
- No automatic promotion from research to Demo or Live.
- No static-universe survivorship shortcut.
- No single-symbol promotion evidence.
- No Live enablement in V13.27.3.
