# AlphaPilot Control Console V13.25.0

V13.25 installs a narrowly allowlisted OKX Live client and a restart-safe
Canary execution ledger.

## Start safely

Read-only mode:

```powershell
powershell -ExecutionPolicy Bypass -File scripts\start_okx_live_canary_console.ps1 -EnableReadOnly
```

Canary process gates, only after release review:

```powershell
powershell -ExecutionPolicy Bypass -File scripts\start_okx_live_canary_console.ps1 -EnableReadOnly -EnableCanary -EnableOrder
```

The second command still does not arm execution. The Console requires a
checksum-matched LiveRelease, active RiskProfile, successful read-only account
reconciliation, and exact `ARM_OKX_LIVE_CANARY` confirmation.

Credentials exist only in the launcher process. The client has no Withdraw
endpoint. Unknown order state, reconciliation mismatch, risk breach, or kill
switch pauses entries. Tests use an injected fake transport and never contact
OKX Live.
