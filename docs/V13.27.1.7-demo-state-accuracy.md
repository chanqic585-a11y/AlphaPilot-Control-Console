# AlphaPilot V13.27.1.7 - Demo Validation State Accuracy

V13.27.1.7 corrects the operator-facing Demo workflow state after real-page
verification.

## Read-only preflight

The runtime-preflight step is complete only when the Demo runtime gates are
ready and the OKX Demo read-only check has passed. A strategy whose next action
is `运行 Demo 前检查` now shows 3/6 (50%), not 4/6 (67%). The card exposes that
action directly and refreshes the workflow after the check.

## Full-market candidate

The current top candidate is sourced only from a real OKX USDT perpetual
full-market scan. When scanning has not started and the instrument count is
zero, the card displays `待扫描` instead of a legacy single-symbol fallback.

## Current-session compatibility

The backend projection is the canonical fix. The frontend also normalizes the
two known stale fields so an already-running console can display the corrected
state without restarting and losing process-only Demo credentials.

## Safety boundary

- The read-only preflight does not place or cancel orders.
- Passing preflight does not automatically start a Demo execution cycle.
- API credentials remain process-only and are never written to the page or
  local data files.
- Requested symbol limits remain capped by strategy and portfolio risk limits.
- Live and withdrawal permissions are unchanged.
