# V13.7.47 - Replay Scoring and Weakness Labels

V13.7.47 turns estimated sandbox replay paths into simple review scores and
weakness labels.

It builds on V13.7.46, which added estimated entry, exit, MFE, MAE, fee,
slippage, and holding time fields from local public OHLCV cache.

## Added Fields

Each replay sample can now include:

- `reviewScore`
- `reviewRating`
- `reviewRatingLabel`
- `primaryWeaknessCode`
- `primaryWeaknessLabel`
- `weaknessLabels`
- `reviewSummary`

Strategy summaries can now include:

- `averageReviewScore`
- `strongReviewSampleCount`
- `usableReviewSampleCount`
- `weakReviewSampleCount`
- `poorReviewSampleCount`
- `topWeaknessLabels`

## Label Examples

- 利润未兑现
- 回撤过深
- 止损型亏损
- 有利波动不足
- 成本拖累
- 持有过久
- 入场路径较干净

## Safety Boundary

Replay scoring is deterministic and explainable. It only interprets local
estimated replay samples.

It is not a strategy signal, not trading advice, not testnet execution, and not
live execution.

V13.7.47 does not store API keys, call Trade API or Withdraw API, read real
accounts or positions, create orders, run exchange Dry-run, or perform
automatic trading.
