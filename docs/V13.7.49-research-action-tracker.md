# AlphaPilot V13.7.49 - Research Action Tracker

V13.7.49 turns the V13.7.48 weakness action board into a local research task
workflow.

## What Changed

The control console now stores action-task state in local `console_state.json`
under `weaknessActionTasks`.

Supported task statuses:

- `todo`
- `in_progress`
- `needs_more_samples`
- `resolved`
- `archived`

## Added API

```text
GET /api/weakness-action-board/tasks
POST /api/weakness-action-task
```

`POST /api/weakness-action-task` accepts:

```json
{
  "actionId": "strategy::weakness_code",
  "taskStatus": "in_progress",
  "note": "local research note",
  "owner": "local_research"
}
```

## Web Console

The strategy weakness action board now includes:

- task status counters
- task-status filter
- priority filter
- start processing button
- needs more samples button
- mark resolved button
- archive button
- local note display

## Safety Boundary

This version is still local research only.

It does not save API keys, connect Trade API, connect Withdraw API, read real
accounts, read real positions, create orders, run exchange Dry-run, or enable
automatic trading.

Marking a research task as resolved does not approve a strategy for trading.
