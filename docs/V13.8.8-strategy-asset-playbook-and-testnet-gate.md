# AlphaPilot V13.8.8 - Strategy Asset Playbook and Testnet Gate

## Goal

V13.8.8 turns the current local sandbox strategy evidence into a readable
strategy asset playbook.

The purpose is to help the operator quickly understand:

- what each candidate strategy is trying to observe;
- which market condition it is designed for;
- how many closed local sandbox samples it has;
- whether it is still collecting evidence, ready for local review, or only a
  testnet preparation candidate;
- why execution remains blocked.

## What Changed

- Added `alphapilot_control_console.strategy_asset_playbook`.
- Added `GET /api/strategy-asset-playbook`.
- Added a Strategy Asset Gate panel to the local web console.
- The panel summarizes strategy count, closed samples, sandbox review
  candidates, testnet candidates, execution lock status, and blocker count.
- The strategy cards show readable family names, one-line explanations,
  sample/PF/R evidence, next action, and testnet blockers.

## Safety Boundary

This version is still local research only.

It does not:

- store API keys;
- add an API key input;
- call Trade API;
- call Withdraw API;
- read real account balances;
- read real positions;
- create real orders;
- execute exchange dry-run;
- enable automatic trading.

## Testnet Gate

The testnet gate remains blocked until the following are designed and verified:

- credential isolation;
- order lifecycle simulator;
- kill switch;
- maximum order and loss limits;
- manual confirmation gate;
- full audit trail.

V13.8.8 only makes the gate easier to see and review. It does not open it.

## Operator Notes

Use the Strategy Asset Gate to decide what to review next:

1. Keep local sandbox running for candidates that still need more samples.
2. Review concentration, data gaps, slippage, and consecutive losses.
3. Do not connect private exchange permissions until the testnet gate is
   explicitly implemented in a later version.
