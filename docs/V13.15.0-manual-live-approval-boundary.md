# V13.15.0 Manual Live Approval Boundary

The Control Console can discover checksum-verified
`live_candidate_package_*.json` files exported by Quant Engine and present them
for local human review.

## State machine

1. `awaiting_manual_approval`
2. `approved_for_future_release_review`
3. `revoked`
4. `checksum_changed_approval_invalid`

Actions are append-only in local SQLite. Approval records bind package ID,
package hash, risk budget, actor, confirmation hash, and UTC timestamp.

Only actor `user_manual` can approve or revoke. Approval requires the exact
phrase `APPROVE_LIVE_CANDIDATE_REVIEW`. AI, Bandit, ML, and automation cannot
write the ledger.

## Non-execution guarantee

Approval does not create an exchange session, read an account, create an order,
or enable a live adapter. The application exposes no live execution endpoint.
Withdraw is unavailable. Raw credentials are not accepted or stored.

The phone calls the local desktop review endpoint only. It never connects to an
exchange and never receives credentials.
