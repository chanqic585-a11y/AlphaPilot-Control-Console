## Runtime risk

The public Runtime Risk contract contains allocated capital, per-trade risk in
percent and USDT, portfolio risk in percent and USDT, concurrent positions,
instrument/direction/correlation-cluster risk, beta, leverage, margin mode,
daily/program/hard-kill limits, and scan depth.

Every change creates an immutable overlay and append-only event. A conservative
change can apply immediately to new orders. A risk increase remains pending
until its exact content hash is manually approved, and it can never exceed the
frozen base profile or disable immutable floors.

## Strategy versions and intervention

Strategy changes create new versions and hashes. New entries can switch while
open positions retain the version that opened them. Flatten-first and manual
position migration are explicit alternatives. Pause, resume, close-only,
rollback, and emergency intervention requests are separately audited.

## Bounded Strategy Factory

The Strategy page can start one bounded generate/combine run across `5m`,
`15m`, `1h`, `4h`, or `1d`. Each run freezes Program/Campaign identity,
comparison panel, candidate budget, trial budget, and safety flags. The Quant
runner receives a pause-marker path so a run can stop cleanly at a checkpoint
and resume without manufacturing progress. The UI first shows
`pause_requested`, then changes to `paused` only after the Quant worker writes
an atomic exit marker. Resume removes the marker and relaunches the same frozen
campaign; it does not create a replacement identity or reuse partial final
artifacts.

Results are projected as: forward-validation needed, Demo eligible, data
insufficient, failed, or system issue. Zero survivors are valid. The factory
cannot force a pass, tune Locked OOS, auto-promote, approve, ARM, or order.
