# OKX Demo 工程烟测操作手册

## 目的

工程烟测只验证 AlphaPilot 与 OKX Demo 的最小技术闭环：认证后的合约发现、最小下单、订单查询、持仓查询、平仓以及最终对账。

它不是策略验证，不计算策略盈亏、胜率或 PF，不产生前向证据，不晋级策略，也不能生成实盘候选。

## 前置条件

1. 使用 OKX Demo Trading 的 Read+Trade 凭据启动控制台。
2. 私有请求必须携带 `x-simulated-trading: 1`。
3. 认证后的 Demo 合约与公共时点资产池必须存在精确 USDT 永续交集。
4. 页面上的 `Demo 工程状态` 必须显示可用，且孤儿持仓数为 0。
5. 只允许在本机回环地址操作；网页不接收、不显示也不保存凭据。

## 执行

在控制台首页的 `Demo 工程状态` 中点击 `运行最小烟测`。每次执行都需要明确确认，启动时不会自动运行。

烟测固定使用：

- 一个认证交集内的 USDT 永续合约。
- 交易所返回的最小下单数量。
- 1x 逐仓。
- 单持仓、禁止加仓、禁止马丁格尔。
- 最多 3 次有界尝试。

成功顺序为：下单尝试 -> 成交查询 -> 持仓确认 -> reduce-only 平仓 -> 空仓对账。

## 结果判断

只有以下条件全部满足才算工程烟测通过：

- 状态为 `completed`。
- 开仓订单状态为 `filled`。
- 平仓状态为 `filled`。
- 最终持仓状态为 `flat`。
- 对账状态为 `passed`。
- 重复订单数为 0。
- 孤儿持仓数为 0。

真实交易所拒单、超时、取消失败或对账不一致都按失败记录，不合成成交，也不伪造持仓。

## 恢复与对账

若页面显示孤儿持仓或对账异常：

1. 暂停再次运行烟测。
2. 点击 `重新对账`。
3. 检查 OKX Demo 中对应合约是否仍有持仓或活动订单。
4. 对账仍失败时，保留独立烟测 SQLite 作为诊断证据，不得把记录迁入策略账本。

烟测账本位于 `data/demo_engineering_smoke.sqlite`。它与策略 Demo、执行结果、晋级、Release 和实盘候选账本物理隔离；代码还会拒绝把烟测写入任何已登记的策略证据路径。

## 安全边界

- 不接 Live API。
- 不接 Withdraw API。
- 不读取或写入实盘账户。
- 不保存 raw API Key、Secret Key 或 Passphrase。
- 不把烟测订单计入策略闭合交易、PF、胜率、PnL、晋级证据、Release 或实盘候选。
- 不允许用工程烟测替代正式回测、Shadow 诊断或 OKX Demo 策略验证。
