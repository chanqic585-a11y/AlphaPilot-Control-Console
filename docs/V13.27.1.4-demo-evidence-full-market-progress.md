# AlphaPilot V13.27.1.4 - Demo Evidence, Full-Market Scan, and Progress

V13.27.1.4 makes the path into OKX Demo visible and configurable without
weakening the Live boundary.

## Permanent evidence checklist

Every Demo strategy always shows the same evidence categories:

- formal backtest evidence;
- target reward/risk ratio of at least 2R;
- complete immutable strategy definition;
- local-forward closed samples;
- formal strategy candidate identity;
- immutable Demo Release;
- process-only OKX Demo runtime readiness;
- closed OKX Demo trades.

The source of each item is also visible: system automatic, manual runtime
operation, or controlled Demo-only override. Missing evidence is never hidden
after a page refresh.

## Controlled Demo-only override

An operator may bypass only the local-forward sample threshold to collect
independent OKX Demo evidence. The action requires a written reason and the
exact confirmation phrase `仅放行到OKX DEMO`.

The override is rejected unless formal backtest evidence exists, the strategy
definition is complete, the target reward/risk ratio is at least 2R, and an
active bounded Demo risk profile exists. The generated release records
`releaseMode=experimental_override` and `livePromotionAllowed=false`.

This override cannot create a Live Candidate, cannot enable Live execution, and
does not bypass any Live release or checksum gate.

## OKX full-market scan

The Demo scanner reads OKX public endpoints only. It discovers live USDT linear
perpetual instruments, validates ticker fields, rejects missing or invalid
prices, filters liquidity and spread, ranks the eligible universe, and performs
a capped deep scan using the frozen strategy family and parameters.

The UI separates:

- the current highest-ranked market candidate;
- the instrument of an actual Demo position;
- universe totals, liquidity-qualified instruments, deeply scanned rows, and
  strategy matches.

No strategy is permanently bound to one symbol by the UI.

## Per-strategy symbol concurrency

Each Demo strategy can request between 1 and 10 simultaneous symbol positions.
The effective limit is the minimum of:

- the strategy preference;
- active Demo RiskProfile positions per strategy;
- active Demo RiskProfile portfolio position limit;
- remaining portfolio and open-risk capacity;
- duplicate-symbol and other execution gates;
- currently matched strategy candidates.

The setting changes capacity only. It does not manufacture a signal or force an
order.

## Progress and compact execution views

Running, queued, and paused backtests show a visible progress track. Known data
download partition counts are displayed exactly; otherwise the track represents
the current workflow phase and does not claim a time estimate.

Demo keeps its six-step progress model and now adds permanent evidence,
full-market status, current positions, realized/unrealized PnL, fees, slippage,
and the strategy symbol limit. Live uses the same compact position presentation
while retaining fewer controls and stricter release gates.

## Safety boundary

- Public universe discovery uses no API key.
- OKX Demo credentials remain process-only.
- Raw credentials are not stored in files, browser state, or SQLite.
- Withdraw remains disabled.
- The Demo override cannot promote to Live.
- Live orders remain protected by immutable candidate/release, RiskProfile,
  reconciliation, process gates, and manual ARM controls.
