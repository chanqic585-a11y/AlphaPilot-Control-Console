# AlphaPilot V13.27.7 - Canonical OKX Instrument Fix

V13.27.7 fixes a Demo full-market scanner defect. Dynamic-universe candidates
already use canonical IDs such as `ETH-USDT-SWAP`, but the public connector
previously normalized that value a second time into the invalid
`ETH-USDT-SWAP-USDT-SWAP`.

The normalizer is now idempotent. `ETH`, `ETHUSDT`, `ETH/USDT`, `ETH-USDT`, and
`ETH-USDT-SWAP` all resolve to exactly `ETH-USDT-SWAP`.

This restores the intended scan sequence:

1. Read the live OKX USDT perpetual universe.
2. Rank liquid contracts.
3. Load valid ticker, candle, spread, and instrument metadata.
4. Evaluate the frozen strategy rules.
5. Apply sizing, portfolio, account, and risk gates before any Demo order.

No strategy threshold is relaxed, including the existing `targetR >= 2`
boundary. Raw credentials remain process-only, Withdraw remains absent, and
Live execution is unchanged.
