# AlphaPilot V13.27.1 - Workflow Strategy Page

V13.27.1 replaces the Strategy page's ambiguous cross-stage counts with one
projection owned by the Quant Engine workflow registry.

## Main lanes

The first viewport shows only:

- 待回测
- 回测中
- 回测通过
- 未通过 / 阻塞
- 已归档策略

Each immutable strategy version appears once. Historical lifecycle summaries
and legacy research assets remain available under `研究档案与旧实验`, but they
do not change the workflow counts.

## Actions

The page can request:

- run one awaiting backtest;
- run all awaiting backtests;
- pause, cancel, or retry a run;
- archive a strategy version;
- advance a passed version to the next stage.

The Control Console does not decide pass or fail. It calls the fixed Quant
Engine CLI and refreshes the read model from `GET /api/workflow`. Worker
progress is polled while work is queued or running.

## Failure presentation

The first viewport uses concise Chinese explanations and recommended next
actions. Internal ids, hashes, raw metrics, and legacy experiments stay in
advanced details so auditability is retained without crowding the page.

## Mobile behavior

At phone width the summary cards and four lanes collapse to one column. Action
buttons become full-width rows, the page has no horizontal overflow, and the
floating `回到策略` command is hidden while the user is already on the Strategy
page.

## Safety boundary

The workflow client launches only an allowlisted local Python module with fixed
arguments and no shell. V13.27.1 does not persist credentials, call Withdraw,
create exchange orders, enable Live execution, or automatically promote a
strategy.
