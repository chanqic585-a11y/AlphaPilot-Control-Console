# V13.8.2 Pre-Live Preparation Pack

V13.8.2 adds a local pre-live preparation pack to the AlphaPilot Control
Console.

This release is still pre-execution. It does not connect to exchange private
APIs, does not accept credentials, does not create testnet orders, does not
create live orders, and does not run exchange Dry-run.

## What Changed

- Added `GET /api/pre-live-preparation-pack`.
- Added `POST /api/pre-live-order-lifecycle/simulate`.
- Added a web panel named `PRE-LIVE PREPARATION PACK`.
- Added a local order lifecycle rehearsal:
  - research candidate selected
  - local intent draft
  - pre-trade risk check
  - manual confirmation ticket
  - simulated submit
  - simulated accept or reject
  - simulated fill or cancel
  - audit closeout
- Added visible risk controls:
  - 1000 USDT virtual capital per strategy
  - 1R / 1% max local rehearsal risk
  - fixed 2R target review requirement
  - -2R daily virtual loss lock
  - one open intent per strategy
  - cooldown after rejection
- Added kill-switch design rows:
  - global execution lock
  - strategy-level pause
  - manual kill switch
  - network authority block
- Added credential-vault design rows:
  - no raw secret storage
  - Withdraw permission forbidden
  - testnet scope first
  - manual unlock

## Reference Absorption

The pack records reference-only lessons from previously saved sources:

- alpha101: factor research workflow inspiration.
- CryptoAgentPro.beta: execution-risk and permission-isolation reference.
- TradingAgents: multi-role review workflow inspiration.
- QuantDigger: event lifecycle and local state-machine inspiration.

No external source code is copied into execution paths.

## Safety Boundary

V13.8.2 keeps these disabled:

- API Key input
- API Key storage
- Trade API
- Withdraw API
- private exchange connection
- real account reads
- real position reads
- testnet order creation
- live order creation
- exchange Dry-run
- automatic trading

The lifecycle simulation endpoint returns a local preview path only. It does
not persist an order and does not contact any exchange.
