# AlphaPilot V13.10.5 - Local Lifecycle Outcome Advancer

## Purpose

V13.10.5 advances the local observation records created by the no-ticket
engine. It converts public market movement into append-only lifecycle events so
the console can accumulate reviewable closed samples without changing the
original audit records.

This is a local research workflow. It is not an exchange execution workflow.

## Data Flow

```text
Local auto-execution record
  -> OKX public ticker and confirmed OHLCV
  -> ATR14 risk-unit calculation
  -> append-only lifecycle event
  -> latest-state projection
  -> lifecycle board and review queue
  -> closed-sample descriptive report
```

## Reference Price Rule

The first available public price becomes the local observation reference. The
record explicitly labels it as a research reference, not a historical fill or
exchange execution price. Existing records without a price are initialized on
their first advance. New records may reuse the price captured by the public
candidate scan.

The first initialization event always starts at `0R` and does not claim a win
or loss.

## Risk and Outcome Rules

- `1R = ATR14 * strategy ATR multiplier`.
- Strategy names such as `ATR1.0` and `ATR1.2` provide the multiplier.
- A missing multiplier falls back to `1.0`; the fallback is recorded.
- Long R: `(current price - reference price) / risk unit`.
- Short R: `(reference price - current price) / risk unit`.
- Target: at least `2R`.
- Stop: `-1R` unless the existing local policy supplies another positive stop
  distance.
- Expiry: 96 bars for 15m, 72 bars for 1h, 42 bars for 4h, and 30 bars for 1d.
- Confirmed candles after the reference time are checked in chronological
  order so a barrier crossed between manual advances is not missed.
- If a single confirmed candle touches target and stop, ordering is unknown;
  V13.10.5 conservatively records the stop and labels the ambiguity.

Each event stores the current price, R value, MFE, MAE, public snapshot summary,
event type, and safety flags. Candle rows are used transiently and raw OHLCV is
not copied into console state.

## Append-Only Projection

`autoExecutionRecords` remains immutable. V13.10.5 adds
`autoExecutionLifecycleEvents`. Read-only screens combine each base record with
its newest event projection. This preserves audit history and allows closed
records to stop blocking future candidates without mutating the original row.

## API

### Advance local lifecycle

```text
POST /api/auto-execution-lifecycle/advance
```

Optional payload:

```json
{
  "maxRecords": 20
}
```

One failed symbol does not block the remaining records. Failed public reads do
not create fake events.

### Read closed-sample learning summary

```text
GET /api/auto-execution-learning
```

The report includes overall, strategy, symbol, and direction breakdowns. Fewer
than 30 closed samples remain descriptive only. Reaching a threshold only
unlocks deeper manual review; it never promotes a strategy automatically.

## Safety Boundary

- Public market data only.
- No API key required or stored.
- No private exchange endpoint.
- No account or position read.
- No Demo or live order.
- No exchange Dry-run.
- No automatic trading.
- No automatic strategy promotion or elimination.
- No model training in V13.10.5.
