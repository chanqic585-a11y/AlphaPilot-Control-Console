# AlphaPilot V13.27.1.6 - Workflow Checkpoint Resume

V13.27.1.6 completes the backtest interruption lifecycle.

## Runtime Rules

- Browser close does not stop the worker.
- A Console or computer restart resumes `queued` and `running` backtests from
  the same workflow run and its persisted checkpoint.
- Explicit pause remains paused. The user must click `继续运行` to resume it.
- Cancelled, failed, blocked, passed, and awaiting runs are not restarted.
- A run-scoped OS file lock prevents duplicate workers after a Console restart.
- Completed official-data partitions are verified and reused; they are not
  downloaded again merely because the process restarted.

The Strategy page keeps the existing progress bar. While the process is down,
the last durable checkpoint remains visible. After the next Console start, the
worker continues and the same progress bar advances when the next checkpoint is
saved.

## Recovery Visibility

`GET /api/health` includes `workflowRecovery` with candidate, started,
already-running, and error counts. Recovery failures do not crash the Console.

## Safety Boundary

- No workflow result is fabricated.
- No strategy version is silently replaced.
- No raw API credential is persisted.
- This patch does not broaden Demo or Live execution permissions.
- Demo 一次输入全部共用；实盘账户一次输入、策略逐条批准启用。
