# V13.14.0 OKX Demo API Contract

核对日期：2026-07-10

本说明只依据 OKX 官方 API 文档和官方 API FAQ，定义 AlphaPilot V13.14.0 的 OKX Demo 接口边界。它不是实盘接入说明。

## 环境和认证

- Demo REST 固定使用 `https://openapi.okx.com`。
- 每个 Demo 私有请求必须携带 `x-simulated-trading: 1`。
- 私有请求使用 `OK-ACCESS-KEY`、`OK-ACCESS-SIGN`、`OK-ACCESS-TIMESTAMP` 和 `OK-ACCESS-PASSPHRASE`。
- 签名原文为 UTC timestamp、HTTP method、带 query 的 request path 和 body 的顺序拼接；签名使用 HMAC-SHA256 后 Base64。
- Demo key 必须从 OKX Demo Trading 区域创建，不能混用实盘 key。
- 凭据只从启动进程环境变量读取，不写入 Git、SQLite、报告、响应或日志。

官方依据：[OKX API Guide - Demo Trading Services and REST Authentication](https://www.okx.com/docs-v5/en/)

## 权限和 IP 白名单

- AlphaPilot Demo 自动执行需要 `Read` 和 `Trade` 权限。
- `Withdraw` 权限始终禁止；客户端没有任何提现、资金划转或充值 endpoint。
- 启动检查调用 `GET /api/v5/account/config`，核对账户模式、持仓模式和 key 权限。
- 强烈建议绑定固定 IP 白名单。没有可控固定出口 IP 时，应保持自动执行关闭。

官方依据：[OKX API Guide - API key permissions and security](https://www.okx.com/docs-v5/en/)

## Allowlist

V13.14.0 客户端只允许：

| 方法 | Endpoint | 用途 |
| --- | --- | --- |
| GET | `/api/v5/account/config` | 账户和权限检查 |
| GET | `/api/v5/account/balance` | Demo 余额核对 |
| GET | `/api/v5/account/positions` | Demo 持仓对账 |
| GET | `/api/v5/trade/order` | 单笔订单恢复和查询 |
| GET | `/api/v5/trade/orders-pending` | 活跃订单恢复 |
| GET | `/api/v5/trade/fills` | 成交审计 |
| POST | `/api/v5/trade/order` | Demo 下单 |
| POST | `/api/v5/trade/cancel-order` | Demo 撤单 |
| POST | `/api/v5/trade/cancel-all-after` | Demo kill switch 防护 |

任何不在 allowlist 的路径在网络请求前即被拒绝。

官方依据：[OKX API Guide - Account and Order Book Trading APIs](https://www.okx.com/docs-v5/en/)

## 幂等和订单恢复

- `clOrdId` 限 1 至 32 位、区分大小写的字母或数字。
- OKX 只要求 `clOrdId` 不与当前挂单重复；历史上可能存在重复值，按 `clOrdId` 查询时只返回最近匹配。
- 因此 AlphaPilot 不能只依赖交易所 ID。每个信号先写本地 SQLite 幂等意图，再发送请求，并保存 OKX `ordId`。
- 超时或响应不明时，状态写为 `unknown`，立即暂停新开仓，然后通过 `ordId` 或 `clOrdId` 查询恢复；不得盲目重发。

官方依据：[OKX API Guide - Place order and Get order details](https://www.okx.com/docs-v5/en/)

## 部分成交和保护退出

- 订单状态至少处理 `live`、`partially_filled`、`filled`、`canceled` 和 `mmp_canceled`。
- IOC 订单取消时仍可能已有部分成交，不能把 `canceled` 等同于零成交。
- V13.14.0 可以随普通订单提交 attached TP/SL，但必须继续核对主订单与保护订单状态，不能把一次 HTTP 成功当成完整保护闭环。
- 本地账本与 Demo 持仓不一致时，停止新开仓并触发 kill switch 审计。

官方依据：[OKX API Guide - Order state and attached TP/SL fields](https://www.okx.com/docs-v5/en/)

## 频率、时钟和错误处理

- Place order 和 Get order details 的官方上限均为每个相关限流维度 60 次/2 秒；AlphaPilot 仍使用更低的本地节流。
- Get positions 为 10 次/2 秒；Get account config 为 5 次/2 秒。
- Cancel All After 为 1 次/秒，允许 `0` 或 `10` 至 `120` 秒。
- `50102` 表示请求时间戳过期。主机 UTC 时钟不同步时必须暂停，不允许通过放宽时间校验绕过。
- 交易相关 REST 与 WebSocket 限额可能共享；未来增加 WebSocket 时必须继续统一限流。

官方依据：[OKX API Guide](https://www.okx.com/docs-v5/en/)、[OKX API FAQ - 50102](https://www.okx.com/en-us/help/api-faq)

## AlphaPilot 固定边界

- 仅 OKX Demo Trading。
- 默认测试权益 1000 USDT。
- 单笔账户风险 0.25%，全部未平仓风险最多 1%。
- 单笔名义价值最多 250 USDT，同时持仓最多 3 个，默认最高 2 倍杠杆。
- 单日亏损达到 2% 或 Demo 回撤达到 5% 时暂停新开仓。
- 只有不可变 `DemoRelease` contract 可以进入自动 Demo 执行。
- 不保存 raw API key，不允许 Withdraw API，不允许实盘 endpoint，不自动形成 Live release。
