# V13.27.1.62.1 Strategy Factory Runtime Profile

The one-click strategy factory can use a local, versioned Development profile at:

`data/strategy_factory/development_profile.json`

Alternatively set `ALPHAPILOT_STRATEGY_FACTORY_DEVELOPMENT_PROFILE` to another local JSON file. The file is not committed because it contains machine-local data paths. A portable template is available at `config/strategy_factory_development_profile.example.json`.

At run creation the profile is validated against its immutable snapshot manifest, hashed, and copied into the frozen job. Explicit request fields must match the configured profile; mixed identities are rejected.

This profile only enables read-only Development replay. It does not approve a candidate, create a Release, ARM Demo or Live, access private account data, or create orders.

The Strategy and factory run projections now reconcile worker receipts before rendering. A finished worker therefore cannot remain displayed as running merely because the page used the single-run or run-list endpoint.
