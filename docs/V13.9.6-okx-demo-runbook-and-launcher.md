# V13.9.6 OKX Demo Runbook and Launcher

## Goal

V13.9.6 makes the OKX Demo stage easier to operate without weakening the safety
boundary.

It adds:

- a temporary environment-variable launcher
- a Demo runbook strip in the web console
- a latest read-only check result panel

It does not add live trading or automatic trading.

## Launcher

Run from `D:\Codex-Workspace\AlphaPilot-Control-Console`:

```powershell
powershell -ExecutionPolicy Bypass -File scripts\start_okx_demo_console.ps1
```

The launcher prompts for:

- OKX Demo API Key
- OKX Demo Secret Key
- OKX Demo Passphrase

The values are placed in the current PowerShell process environment only. They
are not written to a file, browser state, SQLite, or the local console state.

Optional switches:

```powershell
-Mobile
-EnableOrder
-EnableCancel
```

`-EnableOrder` and `-EnableCancel` are off by default.

## Demo Page

The OKX Demo page now shows:

- startup command
- runbook steps
- credential status
- connection status
- latest read-only check result
- blockers
- recent Demo events

## Read-only Check

The read-only check verifies OKX Demo account balance and SWAP position endpoints
with:

```text
x-simulated-trading: 1
```

If credentials or gates are missing, the check is saved as blocked and no OKX
private request is sent.

## Order Gate

Demo order rehearsal still requires:

- `ALPHAPILOT_OKX_DEMO_ENABLED=1`
- `ALPHAPILOT_OKX_DEMO_ORDER_ENABLED=1`
- complete Demo credentials
- explicit OKX `sz`
- page confirmation phrase `OKX_DEMO_ORDER_APPROVED`
- notional amount at or below 1000 USDT

## Safety Boundary

V13.9.6 does not enable:

- live trading
- live API keys
- Withdraw API
- automatic trading
- real account trading
- real order creation

The active path remains:

```text
Local Sandbox -> Local Simulation -> OKX Demo -> manual review -> future live gate
```
