# AlphaPilot V13.8.1 - Testnet Readiness UI Boundary

V13.8.1 adds a disabled Testnet readiness center to the local Control Console.

This is a design boundary release. It does not connect to exchange testnet,
does not save credentials, and does not create orders.

## What Changed

- Added `GET /api/testnet-design-boundary`.
- Added a greyed Testnet readiness center in the console.
- Added disabled action buttons for:
  - API Key input
  - Testnet connection
  - Testnet order creation
  - Exchange Dry-run
- Added a future connection sequence:
  - design freeze
  - credential vault specification
  - order lifecycle simulator
  - kill switch and limits
  - manual confirmation gate
  - future read-only testnet probe

## Why This Exists

The user wants the project to be ready for future testnet and live trading, but
the system is not allowed to jump directly from local sandbox research into
exchange execution.

This page makes the missing controls visible before any trading capability is
implemented.

## Safety Boundary

V13.8.1 does not add:

- API key storage
- API key input
- Trade API
- Withdraw API
- real account reads
- real position reads
- real order creation
- testnet order creation
- exchange Dry-run
- live trading
- automatic trading

The console remains a local research and sandbox observation tool.
