# AlphaPilot V13.6.3 - Mobile Connection Helper

V13.6.3 is a usability and verification patch for the AlphaPilot Control Console.

It does not add trading permissions.

## Goal

Make the desktop Control Console easier to connect from the mobile App during real-device testing.

## Added

- `GET /api/mobile/connection-info`
- LAN URL discovery for likely phone-accessible desktop addresses
- Desktop UI display for the recommended phone URL
- `scripts/smoke_mobile_bridge.ps1`
- README instructions for phone connection testing

## Real Phone Testing

`127.0.0.1` points to the phone itself on Android. A phone must use the desktop PC LAN address.

Example:

```text
http://192.168.x.x:8766/api/mobile/status
```

The helper endpoint lists candidates:

```text
http://127.0.0.1:8766/api/mobile/connection-info
```

If the phone cannot connect:

- Confirm phone and desktop are on the same Wi-Fi or LAN.
- Start the Control Console with a LAN-visible host:

```powershell
powershell -ExecutionPolicy Bypass -File scripts\start_console.ps1 -Mobile
```

- Allow Python through Windows Firewall for port `8766`.
- Confirm `/api/mobile/status` works on the desktop first.

## Smoke Test

```powershell
powershell -ExecutionPolicy Bypass -File scripts\smoke_mobile_bridge.ps1
```

The smoke test checks:

- `/api/health`
- `/api/mobile/status`
- `/api/mobile/connection-info`
- execution locks remain disabled

## Safety Boundary

V13.6.3 remains read-only.

It does not:

- save API keys
- add API key input
- call private exchange endpoints
- use Trade API
- use Withdraw API
- read real account balances
- read real positions
- create orders
- run exchange dry-run
- trade live
- trade automatically
