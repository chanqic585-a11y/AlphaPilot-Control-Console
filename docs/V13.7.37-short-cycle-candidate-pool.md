# AlphaPilot V13.7.37 - Short Cycle Candidate Pool

V13.7.37 adds a short-cycle research candidate pool to the desktop control
console.

This version does not add real high-frequency trading infrastructure. In this
project, "short cycle" means 15m / 30m / 1h public OHLCV research candidates.
It does not mean tick-level, order-book, colocated, or seconds-level HFT.

## Candidate Pool

The console selects five candidates:

1. 15m Volume Rebound
2. 1h Trend Pullback
3. 1h Short Rejection
4. 30m Volatility Compression Breakout
5. 30m Bollinger Mean Reversion

The first three candidates are linked to existing short-cycle or short-cycle
adjacent assets. The last two are derived from existing higher-timeframe
strategy assets and must be independently tested on 30m data before they can be
used for observation.

## API

```text
GET /api/short-cycle-candidates
```

The endpoint returns:

- summary counts
- target timeframes
- selected candidate rows
- source asset metadata
- short-cycle score
- missing metrics
- entry idea
- risk focus
- next validation action
- safety notes

## Validation Boundary

The pool is for research scheduling only. It is not a signal engine and it does
not create orders.

Before any candidate can move beyond this pool, it must pass:

- public OHLCV backtest
- fee and slippage stress
- liquidity gate
- BTC regime filter
- market-state breakdown
- forward observation logs
- manual review

## Safety Boundary

V13.7.37 keeps all execution capability disabled:

- no API key input
- no Trade API
- no Withdraw API
- no account access
- no position access
- no real orders
- no exchange Dry-run
- no live trading
- no automatic trading
