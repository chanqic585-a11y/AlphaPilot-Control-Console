# V13.9.9 Control Console Performance Patch

V13.9.9 is a runtime polish patch for the AlphaPilot desktop control console.
It does not add strategy logic, exchange trading, live trading, API key storage,
or automatic execution.

## Problem

The console could feel slow even on localhost because the first page load pulled
large and expensive read-only payloads:

- local sandbox result review
- local sandbox quality center
- simulation bridge and simulation review
- strategy learning loop
- full mobile status JSON
- advanced research and testnet panels

The latency came from local report scanning and large JSON payloads, not from
network speed.

## Changes

- First render now loads only the core console summaries needed by the five main
  pages.
- Local sandbox review data is loaded after the first render.
- Local simulation and learning-loop data are loaded when the local simulation
  page is opened.
- Full mobile status is loaded only when the mobile console page is opened.
- Advanced research data is loaded only after the advanced research section is
  expanded.
- Slow read-only GET routes now use short in-process TTL caches.

## Cached GET Routes

The cache is intentionally short-lived and only applies to read-only route
builders. POST routes still execute normally.

Examples:

- `/api/local-sandbox/result-review`
- `/api/local-sandbox/quality-center`
- `/api/simulation-review`
- `/api/simulation-bridge`
- `/api/strategy-learning-loop`
- `/api/mobile/status`
- `/api/exchange-demo/simulation`
- `/api/strategy-asset-playbook`

## Safety Boundary

This patch does not enable:

- Trade API
- Withdraw API
- raw API key storage
- real account access
- real position access
- real order creation
- automatic trading

The console remains a local-first research, sandbox, and gated Demo control
surface.
