# AlphaPilot V13.10.3 - Local Auto Execution Lifecycle Monitor

V13.10.3 adds a lifecycle monitor for V13.10.2 local no-ticket auto execution
records.

## Scope

- Add a read-only lifecycle API: `GET /api/auto-execution-lifecycle`.
- Add a desktop console lifecycle board.
- Group local records into clear stages:
  - waiting trigger
  - local simulated holding
  - 2R take-profit
  - -1R stop-loss
  - expired exit
  - blocked
- Surface router and risk-gate blockers in the UI.

## Safety Boundary

This version only reads local simulation and audit records.

It does not:

- store raw API keys
- connect private exchange endpoints
- read real accounts
- read real positions
- submit Demo orders
- submit live orders
- enable live automatic trading

## Data Meaning

Lifecycle rows are local state projections from `autoExecutionRecords`.

They are not exchange positions, not realized PnL, and not proof that a strategy
is ready for live trading. They are review records that help decide which
strategies need more observation, risk review, or Demo-stage rehearsal.

## UI

The Demo page now shows a lifecycle board below recent auto-execution records.
It is designed to answer:

- Which strategies are locally active?
- Which records were blocked?
- Which records reached local 2R / -1R / expired states, when those statuses
  are present in local records?
- Why did a strategy fail the router or risk gate?
