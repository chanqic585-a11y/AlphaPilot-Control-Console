# V13.9.0 to V13.9.1 - Testnet Read-only Check and Small Order Simulation

## Scope

V13.9.0 adds a local Testnet read-only permission check.

V13.9.1 adds a local-only small Testnet order simulation ticket.

These releases do not connect private exchange endpoints and do not create
exchange orders.

## Added APIs

```text
GET  /api/testnet-permission-check
GET  /api/testnet-small-order-simulation
POST /api/testnet-small-order-simulation/rehearse
```

## V13.9.0 Permission Check

The permission check combines:

- existing public exchange probe status
- API key disabled status
- private testnet connection disabled status
- Trade API and Withdraw API disabled status
- manual unlock blocker
- reference-only design inputs

The check can say the console may continue to local small-order simulation, but
it cannot approve private Testnet connection.

## V13.9.1 Small Order Simulation

The small-order simulation uses:

- virtual account: 1000 USDT
- max local ticket size: 5 USDT
- max local risk: 0.1R
- manual simulation approval: required

The resulting record is saved locally as a simulation ticket. It is not an
exchange order.

## Reference Inputs

Previously saved references are used only as design metadata:

- alpha101: factor research workflow and explainable feature context
- CryptoAgentPro.beta: permission separation, risk gateway, and emergency
  boundary reference
- TradingAgents: multi-role review and structured checkpoint reference
- QuantDigger: event lifecycle and state-machine reference

No external execution code is copied into the control console.

## Safety Boundary

V13.9.0 and V13.9.1 do not:

- accept API keys
- store raw API keys
- connect private exchange endpoints
- use Trade API
- use Withdraw API
- read real accounts
- read real positions
- create exchange orders
- run exchange Dry-run
- run live trading
- run automatic trading

The current implementation is a local control-console rehearsal layer only.
