# V13.7.46 - Sandbox Sample Path Instrumentation

V13.7.46 upgrades the closed-sample replay panel from outcome-only records to
estimated path review records.

The console now uses local public OHLCV cache files to estimate:

- entry time and price
- exit time and price
- research direction
- market regime
- MFE and MAE in R
- path outcome in R
- fee and slippage estimates
- holding time
- replay candle window

These fields are added to newly generated local sandbox samples and are also
computed dynamically for older logs when `GET /api/closed-sample-replay` is
called.

## Important Boundary

The path is estimated from local public OHLCV cache. It is not an exchange fill,
not testnet execution, and not live execution.

Every enriched sample keeps:

```text
actualExchangeFill = false
instrumentationMode = estimated_from_local_ohlcv_cache
```

The replay panel remains local research only. It does not store API keys, call
Trade API or Withdraw API, read real accounts or positions, create orders, run
exchange Dry-run, or perform automatic trading.

## Why This Matters

V13.7.45 made closed samples visible but many old samples lacked enough detail
for practical review. V13.7.46 makes the sample more useful without pretending
it is a real trade record.

This prepares the next review layer:

- inspect why a strategy won or lost
- compare MFE versus MAE
- separate cost drag from path behavior
- identify strategies that need better entry timing or tighter filtering
- keep the upgrade gate conservative before any future testnet design
