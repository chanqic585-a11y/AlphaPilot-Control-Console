# AlphaPilot V13.7.45 - Closed Sample Detail and Trade Replay View

V13.7.45 adds a read-only closed-sample replay view to the local Control
Console.

## Purpose

The goal is to make local sandbox closed samples easier to review before any
future testnet design. It shows what happened in each representative sample and
what data is still missing for a true trade-path replay.

## API Endpoints

- `GET /api/closed-sample-replay`
- `GET /api/closed-sample-replay/strategies`
- `GET /api/closed-sample-replay/samples`
- `GET /api/closed-sample-replay/strategies/{strategyId}`

## Data Rules

- Closed sample counts come from the latest local sandbox daily report
  `closedPaperSampleCount`.
- Raw repeated observation logs are not used to inflate sample counts.
- Samples are representative local replay rows, not real fills.
- If the local log lacks full path fields, the UI shows them as pending.

## Current Missing Replay Fields

- `entryTime`
- `exitTime`
- `entryPrice`
- `exitPrice`
- `direction`
- `marketRegime`
- `mfeR`
- `maeR`
- `feeEstimate`
- `slippageEstimate`
- `holdingTimeMinutes`

## Safety Boundary

V13.7.45 does not store API keys, connect Trade API, connect Withdraw API, read
real accounts, read real positions, create orders, run exchange Dry-run, or
enable automatic trading.

The feature is local research only.
