# AlphaPilot V13.27.6 - Runtime Resume and Official Data Progress

V13.27.6 fixes two operator-visible stalls without changing strategy logic or
trading boundaries.

## Demo runtime resume

An explicit Demo start clears an ordinary paused flag before the runtime is
armed. It never clears the kill switch. The next heartbeat still runs process
credential, read-only reconciliation, account, order-state, automation, and
risk checks. A genuine failure pauses Demo again.

## Official data progress

The strategy workflow keeps its overall phase progress and additionally shows
the active OKX partition, downloaded page count, K-line count, and partition
percentage. The page checkpoint is durable operator telemetry; partial rows do
not count as a completed data partition or formal backtest evidence.

## Safety boundary

- Runtime credentials remain process-only.
- No raw API key is written to files or SQLite.
- Withdraw is absent.
- Live execution is not resumed or enabled by this patch.
- Formal promotion still requires `targetR >= 2` and existing evidence gates.
