# AlphaPilot V13.6 - Control Console Bridge Design

## Goal

Build a local desktop web control console that can read AlphaPilot Quant Engine
research outputs and expose a mobile-safe status bridge.

V13.6 prepares the product shell for future strategy import and approval. It
does not execute trades.

## Positioning

```text
Desktop web console = review, import, monitor, approve later
Mobile app bridge = read-only control status
Quant Engine = strategy research and report generation
```

## Safety Boundary

V13.6 does not:

- store API keys
- request API keys
- call Trade API
- call Withdraw API
- read real accounts
- read real positions
- create orders
- run exchange Dry-run
- enable live trading
- enable automatic trading

## Architecture

```text
AlphaPilot-Control-Console/
  alphapilot_control_console/
    config.py
    importer.py
    state_store.py
    http_app.py
  web/
    index.html
    styles.css
    app.js
  scripts/
    start_console.ps1
  data/
    console_state.json
    audit_log.jsonl
```

The server is a Python standard-library HTTP server. It binds to
`127.0.0.1` by default and reads local Quant Engine report files from
`D:\Codex-Workspace\AlphaPilot-Quant-Engine`.

## Data Flow

1. The console scans Quant Engine reports.
2. It builds a compact strategy package list.
3. It stores local console decisions in `data/console_state.json`.
4. It writes local audit events to `data/audit_log.jsonl`.
5. It exposes `/api/mobile/status` as a read-only mobile bridge.

## Strategy Status Model

Allowed statuses:

```text
research_only
local_paper_ready
forward_testing
dry_run_candidate
disabled
```

Only local status labels are changed. No order, dry-run, account, or position
action is possible in V13.6.

## API

```text
GET  /api/health
GET  /api/strategies
GET  /api/reports
GET  /api/mobile/status
GET  /api/audit
POST /api/import
POST /api/strategy-status
```

`POST /api/import` rescans local reports only.

`POST /api/strategy-status` updates local console state only.

## UI

The first desktop UI contains:

- system status
- safety boundary banner
- strategy package table
- latest report cards
- mobile bridge endpoint
- audit log

## Acceptance

V13.6 is complete when:

1. The desktop console runs locally.
2. Quant Engine strategy/report files are listed.
3. Mobile bridge JSON is available.
4. Local status changes write audit logs.
5. No trading capability exists.
6. The repo has a clean commit and tag.

## V13.6.1 - Public Exchange Connectivity and Strategy Slot Registry

V13.6.1 extends the control console with a public-only exchange connectivity
layer and a strategy slot registry.

### Exchange Connectivity Boundary

The exchange layer is limited to public market-data probes:

- ticker
- OHLCV
- funding rate when a public endpoint is available
- open interest when a public endpoint is available

The exchange layer does not:

- request API keys
- store API keys
- call private endpoints
- call Trade API
- call Withdraw API
- read accounts
- read positions
- create orders
- cancel orders
- run exchange Dry-run
- enable live trading
- enable automatic trading

### API Additions

```text
GET  /api/exchanges
GET  /api/strategy-slots
POST /api/exchanges/probe-public
```

`POST /api/exchanges/probe-public` accepts a symbol, timeframe, and small candle
limit, then probes OKX, Binance USD-M Futures, and Bybit USDT perpetual public
market data. Results are written to `data/exchange_probe_results.json`, which is
runtime state and is not committed.

### Strategy Slot Registry

The registry defines one active candidate slot, one observer slot, and three
empty backup slots. Slots are local metadata containers and do not execute
strategies.

### Mobile Bridge

The read-only mobile bridge now includes latest public exchange probe status
when a probe has been run. It still exposes status only, not trading controls.
