# AlphaPilot V13.27.1.5 - One-Time Guidance and Safe Demo Launcher

## Product Decision

The credential and approval model is deliberately split by environment:

```text
Demo 一次输入全部共用
实盘账户一次输入、策略逐条批准启用
```

For OKX Demo, the user enters API Key, Secret Key, and Passphrase once when a
new Console runtime starts. Every eligible immutable Demo Release can use that
same process-level account connection. Strategy orders, positions, PnL,
reconciliation, and evidence remain isolated by strategy.

For Live, account credentials are also account-level, but each strategy needs
its own explicit approval and immutable Release/RiskProfile binding. This patch
does not enable Live trading or remove any Live safety gate.

## One-Time Guidance

- Strategy, Local Simulation, Demo Simulation, and Live pages map blockers to
  an actionable issue model.
- Only the highest-priority unresolved issue is shown automatically.
- Acknowledgement uses a stable page, stage, blocker, and version fingerprint.
- The same issue does not repeatedly interrupt the user after acknowledgement.
- `查看处理办法` reopens an acknowledged issue manually.
- The permanent Demo evidence checklist remains available in a collapsed
  section.

## Safe Local Launcher

`POST /api/local-control/open-okx-demo-launcher` accepts no credential fields
and no command arguments. It is available only from a loopback client and opens
a fixed PowerShell script in a visible terminal window.

The script:

1. securely prompts for the three OKX Demo credentials;
2. requires the exact automation confirmation phrase;
3. verifies the expected Console PID, listener port, and Python command line;
4. stops only that verified Console process;
5. starts the replacement Console with Demo order and automation gates;
6. clears credential environment variables when the process exits.

Mobile and LAN clients are directed to open `http://127.0.0.1:8766` on the
Console computer. They cannot start local processes.

## Safety Boundary

- No credential is rendered in HTML, returned by the launcher endpoint, placed
  in command arguments, or persisted by the Console.
- Demo account sharing does not merge strategy ledgers or bypass immutable
  Release and risk gates.
- Live credentials are not collected by the Demo launcher.
- Live strategy approval remains per strategy.
- Withdraw remains disabled.
- No database migration is introduced.
