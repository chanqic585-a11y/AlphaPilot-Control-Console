# V13.9.7 Page-separated Console and Demo Candidate Pipeline

V13.9.7 improves the AlphaPilot Control Console information architecture and
OKX Demo preparation flow.

## Page Separation

The main console is now split into four primary pages:

1. `策略`
2. `本地模拟`
3. `Demo模拟`
4. `实盘交易`

The implementation uses hash-based page routing inside the same local web app.
This keeps the console state stable and avoids opening separate browser windows.

Advanced research modules still exist, but they stay behind the advanced mode
toggle so the main workflow is not crowded.

## Local Simulation Fix

The `本地模拟` page now renders the existing:

- simulation bridge summary
- simulation review queue
- strategy learning-loop summary
- local sandbox daily report
- local sandbox quality center

This fixes the earlier empty-card problem where the API data existed but was not
passed into the page renderer.

## OKX Demo Candidate Pipeline

The `Demo模拟` page now supports this staged flow:

```text
load usable strategies
-> build candidate symbols
-> scan OKX public market data
-> fill the Demo ticket
-> keep the order gate/manual confirmation in place
```

The candidate scan uses public market data only. It does not use private OKX
endpoints, API keys, account balance, positions, or order routes.

## Safety Boundary

V13.9.7 does not enable:

- live trading
- Withdraw API
- raw API key storage
- real account access
- real position access
- real order creation
- automatic trading
- automatic Demo order submission

The Demo ticket fill action only writes values into the local form. Submitting a
Demo order still requires environment-only Demo credentials, the separate order
gate, an explicit `sz`, and the manual confirmation phrase.
