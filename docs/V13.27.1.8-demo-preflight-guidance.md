# AlphaPilot V13.27.1.8 - Demo Preflight Guidance

V13.27.1.8 makes a failed OKX Demo Read-only preflight visible and actionable.

## Current 50110 behavior

When the account-config, balance, or position request returns OKX business code
`50110`, the console records an explicit blocker and keeps the strategy at step
4 of 6. HTTP 200 only means OKX received the request; it does not mean the
private Demo request passed.

The operator guidance asks the user to verify:

1. The key was created in the OKX Demo trading API page and is a transaction
   type Demo key.
2. The current public IP matches the key's IP allowlist. Updating the allowlist
   is preferred over leaving it disabled long term.
3. The API domain matches the account region when an EEA, US, or AU domain is
   required.
4. The process is restarted and process-only credentials are entered again
   after the key, allowlist, or domain changes.

## One-time dialog

The failure is registered with the existing issue-guidance controller. It opens
once for the stable blocker fingerprint, and closing it stores an acknowledgement
in local/session storage. The inline status and event details remain visible.

## Safety

- The preflight reads Demo account configuration, balance, and positions only.
- It does not place or cancel an order.
- Raw credentials remain process-only.
- Live and Withdraw remain disabled.
