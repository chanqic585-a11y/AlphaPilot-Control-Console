# AlphaPilot V13.27.4 - Workflow Recovery and Demo Release

V13.27.4 makes the strategy workflow restart-safe and adds an explicit audited
path from local forward observation to OKX Demo.

## Serial Recovery

On Console startup, incomplete backtests are grouped into one deterministic
serial batch. Explicitly paused tasks are not resumed automatically. All
selected tasks enter the queue before the first heavy worker starts.

## Controlled Demo Release

An eligible local-forward card exposes `人工放行到 Demo`. The action requires a
reason and the exact confirmation phrase `仅放行到OKX DEMO`. It still requires:

- formal backtest trades,
- a complete immutable strategy definition,
- `targetR >= 2`, and
- an active versioned OKX Demo RiskProfile.

The contract records `local_forward_samples` as bypassed evidence and sets:

```text
postDemoPromotionPolicy=demo_validation_supersedes_local_forward_evidence
```

The action does not mark local forward as passed and does not create an order.
The release remains `livePromotionAllowed=false`. Only after complete OKX Demo
validation may the strategy enter live-candidate review, where all separate
Live credentials, risk, reconciliation, ARM, TP/SL, circuit-breaker and stop
gates still apply.

## Credentials

OKX Demo credentials remain process-only. Raw keys are not written to files or
SQLite, and Withdraw is not implemented.
