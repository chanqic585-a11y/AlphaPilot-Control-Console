# AlphaPilot V13.10.0 - No-Key Pre-Live Workbench

V13.10.0 adds a no-key pre-live workbench to keep preparation moving before OKX
Demo credentials are available.

## Purpose

The workbench answers three questions without private credentials:

- Which usable strategies are currently in the catalog?
- Which strategy/symbol pairs have public market data available?
- Which candidates should be saved as local observation tickets before OKX Demo
  read-only checks?

## Local Endpoints

- `GET /api/no-key-pre-live`
- `POST /api/no-key-pre-live/scan`
- `POST /api/no-key-pre-live/create-ticket`

## Safety Boundary

This module is public-market and local-state only.

It does not:

- require API keys
- store raw API keys
- call private exchange endpoints
- read real accounts
- read real positions
- create Demo orders
- create live orders
- enable automatic trading

Observation tickets are local review records only. They are not exchange orders.

## Handoff To OKX Demo

When OKX Demo credentials are ready, start the console from the repository root:

```powershell
powershell -ExecutionPolicy Bypass -File scripts\start_okx_demo_console.ps1
```

Enter the Demo credentials only in the local PowerShell prompt. The no-key
workbench output can be used as context for the later OKX Demo read-only check
and manual Demo rehearsal.
