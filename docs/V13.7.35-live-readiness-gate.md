# AlphaPilot V13.7.35 - Live Readiness Gate

V13.7.35 adds a pre-live readiness gate to the local Control Console.

This version does not add live trading. It does not connect Trade API, Withdraw
API, API keys, real account data, real positions, real orders, exchange Dry-run,
or automatic execution.

## Purpose

The gate answers one question:

```text
Which candidate strategies are close enough to create a local manual-review ticket?
```

It does not answer:

```text
Should the system trade now?
Should the system create an order?
Should the system connect to an exchange account?
```

## Readiness Inputs

The gate reads the existing local Quant Engine scan payload and learning-loop
state:

- Paper observation tasks.
- Observation quality rows.
- Local sandbox closed samples.
- Historical metrics when present.
- Saved local manual tickets.

No exchange request is made by this gate.

## Gate Thresholds

The first version uses conservative local thresholds:

- Minimum observation quality score: 60.
- Minimum observation logs: 10.
- Minimum rule-matched observations: 3.
- Minimum closed samples: 5.
- Minimum historical trades: 30.
- Minimum profit factor: 1.25.
- Minimum reward/risk: 1.8.
- Maximum drawdown: 25%.
- Maximum risk warnings: 1.
- Maximum unresolved invalidations: 0.

The gate also blocks manual ticket readiness before:

```text
2026年7月10日（北京时间）
```

## Manual Execution Tickets

Manual execution tickets are local review records. They store:

- strategy/task identity
- readiness score
- quality context
- historical metric context
- 2R risk-plan reminder
- manual checklist
- safety boundary flags

They are not orders. They do not contain exchange credentials. They do not call
any exchange endpoint.

## API Endpoints

```text
GET  /api/live-readiness
GET  /api/manual-execution-tickets
POST /api/manual-execution-ticket
```

`POST /api/manual-execution-ticket` returns `409 manual_ticket_not_allowed` when
the candidate has not passed the local gate.

## Safety Boundary

V13.7.35 keeps all execution permissions disabled:

- No API key storage.
- No Trade API.
- No Withdraw API.
- No account balance read.
- No position read.
- No order creation.
- No exchange Dry-run.
- No live trading.
- No auto trading.
