# AlphaPilot V13.27.1.3 - Unified Demo Workflow Progress

## Purpose

V13.27.1.3 makes the Strategy, Local Forward, and OKX Demo process visible and
operator-actionable. It does not loosen the immutable Release gate and does not
add Live execution.

## Exclusive Demo queues

```text
待 Demo 模拟 -> Demo 验证中 -> Demo 模拟通过 -> 实盘候选
```

- `待 Demo 模拟`: assigned to Demo Trial but no immutable executable Release.
- `Demo 验证中`: immutable Release exists and Demo evidence is being collected.
- `Demo 模拟通过`: closed-trade and risk review has formally passed.
- `实盘候选`: immutable Live Candidate package exists; Live remains separately
  gated and default-off.

A strategy appears in exactly one queue. Historical backtest and local-forward
evidence remain preserved but do not create duplicate cards.

## Visible process

Each Demo card displays:

1. strategy entered the Demo queue;
2. public market and instrument check;
3. immutable Demo Release;
4. credentials, read-only connection, order gate, automation gate, and risk
   preflight;
5. OKX Demo signal, order, and position lifecycle;
6. closed-trade evidence review.

The page also displays actual instrument, entry/mark/TP/SL values, quantity,
realized/unrealized PnL, fees, slippage, and reconciliation state when the
underlying execution evidence exists. Unknown values render as `--` or
`尚未开始`; backtest metrics are never presented as Demo PnL.

## Current ten strategies

The ten legacy strategies moved from the local page are Demo Trial assignments.
They currently have no formal StrategyCandidate in the Quant registry, no
immutable DemoRelease, and no Demo execution record. The correct current count
is therefore:

```text
待 Demo 模拟: 10
Demo 验证中: 0
Demo 模拟通过: 0
实盘候选: 0
```

The first card action is a public market check. The second action is a Release
readiness check that reports missing evidence. It does not fabricate a Release
or submit an order.

## API

```text
GET  /api/demo-workflow
POST /api/demo-workflow/action
```

Supported stage actions:

- `scan_public_market`
- `prepare_demo_release`
- `run_demo_preflight`
- `run_demo_cycle`
- `retry_demo_cycle`

`run_demo_cycle` resolves the exact Release for the selected strategy and then
reuses the existing checksum-gated automatic Demo engine. Externally supplied
signals are not accepted.

## Safety boundary

- OKX Demo only.
- No raw credential storage.
- No Release bypass.
- No Withdraw API.
- No automatic Live promotion.
- No fabricated order, position, or PnL data.
