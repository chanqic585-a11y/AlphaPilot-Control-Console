# V13.10.4 本地自动执行复核队列

## 目标

V13.10.4 读取 V13.10.3 已保存的本地自动执行生命周期记录，将它们整理成可解释、可复核的中文队列。它只做聚合和展示，不修改原始记录，不自动晋级或淘汰策略。

## 新增内容

- 新增 `alphapilot_control_console/auto_execution_review.py`。
- 新增只读接口 `GET /api/auto-execution-review`。
- 新增生命周期复核摘要、阻塞原因分布、活跃持有队列、闭合结果队列、阻塞复核队列和策略生命周期汇总。
- 新增按币种、方向的生命周期拆分。
- 新增样本门槛和中文系统建议。
- 新增复核优先级和阻塞原因标准化覆盖率。
- Demo 页面新增“本地自动执行复核队列”面板。

## 数据解释

阻塞原因从原记录的 `routerBlockers`、`riskBlockers` 以及安全的说明字段中提取。可识别原因会映射为中文分类；无法可靠识别时统一显示“未知原因”，不会推测具体根因。

每条阻塞记录只选择一个主要原因用于分布统计，因此阻塞原因占比合计为 100%。其他已识别原因保留在记录的阻塞说明中。

活跃持有记录没有入场价、当前价或 R 值时，对应字段保持 `null`，页面显示缺失警告，不用零值或默认盈利替代。

## 当前样本门槛

- 每条策略阻塞原因样本达到 10 条后，可以分析阻塞质量。
- 每条策略闭合结果达到 30 条后，可以初步评估。
- 每条策略闭合结果达到 100 条后，可以严肃复核。
- 每条策略闭合结果达到 300 条后，可以做稳定性判断。

当前仍属于执行链路观察阶段，不是策略晋级阶段。

## 安全边界

- 不保存真实 API Key。
- 不接 Trade API 或 Withdraw API。
- 不读取真实账户或持仓。
- 不创建 Demo、Testnet 或实盘订单。
- 不执行 exchange Dry-run。
- 不自动交易。
- 不自动晋级或淘汰策略。
