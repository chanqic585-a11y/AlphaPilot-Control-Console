# AlphaPilot V13.27.9 - Top100 Demo Release and Low-Latency Public Runtime

## Purpose

V13.27.9 shortens the path from an OKX confirmed candle close to an eligible
OKX Demo order while preserving the existing release, risk, and environment
boundaries. It is not a strategy-rule relaxation and it does not enable Live.

The public market runtime covers the OKX USDT linear perpetual market, applies
the existing liquidity filter, and keeps the ranked Top100 pool warm for every
timeframe required by the active immutable Demo releases.

## Runtime Design

- Tickers use the unauthenticated OKX public WebSocket endpoint.
- Candlesticks use the unauthenticated OKX business WebSocket endpoint.
- REST is used only to seed or reseed public state before readiness.
- The hot scanner reads one frozen in-memory snapshot per timeframe.
- Factors are precomputed when confirmed market state changes, not once per
  strategy.
- A timeframe batch is emitted only after all required Top100 instruments have
  the same confirmed candle.
- The 15-second controller heartbeat performs health/reconciliation work only;
  it cannot reconstruct a missed confirmed-close order cycle.

No public runtime payload accepts credential-like fields. API Key, Secret Key,
and Passphrase remain process-only in the secure Demo launcher.

## Immutable Top100 Successors

V13.27.9 does not edit an active immutable Top20 release. The migration creates
a new contract identity, content hash, and contract hash with:

```text
mode = okx_usdt_linear_perpetual_full_market
screeningLimit = 100
policyVersion = okx_full_market_policy_v2_top100
```

The predecessor bytes and migration manifest are archived. The SQLite runtime
is backed up before activation, only predecessor checkpoints are retired, and a
mid-swap failure restores both contracts and database state.

## Latency Policy

The operator target is confirmed close to order send within five seconds.

| Close age | Decision |
| --- | --- |
| `0-5s` | Normal eligible entry |
| `5-10s` | Delayed but still eligible |
| `10-30s` | Conditional late entry only |
| `>30s` | Expired, no order |

A conditional late entry additionally requires:

- Quote age no greater than two seconds.
- Liquidity, spread, immutable release, and portfolio risk gates still pass.
- Adverse price drift no greater than the smaller of `0.20%` and `10%` of the
  original stop-distance percentage.
- Fee/slippage-adjusted net reward/risk remains at least `2R`.

The console shows redacted durations for close-to-evaluation, evaluation,
strategy arbitration, risk checks, order send, exchange response, and latency
class. It does not expose Top100 raw quotes, account details, or credentials.

## No-Order Rehearsal

Create the pinned workspace runtime once:

```powershell
powershell -ExecutionPolicy Bypass -File scripts\setup_console_runtime.ps1
```

Run the public-only no-order rehearsal:

```powershell
powershell -ExecutionPolicy Bypass -File scripts\rehearse_top100_latency.ps1
```

The JSON report is written to the ignored path:

```text
data/ops/v13_27_9_top100_latency_rehearsal.json
```

It reports P50/P95/max scan latency and must always contain:

```text
privateCallCount = 0
orderCallCount = 0
createsDemoRelease = false
changesRuntimeState = false
```

Rehearsal evidence measures local scanner performance only. It does not prove
real WebSocket close coverage or real exchange response latency.

## Controlled Activation

Before activation, stop Demo automatic execution and verify
`desiredEnabled = false`. Then run:

```powershell
powershell -ExecutionPolicy Bypass -File scripts\activate_top100_demo_releases.ps1
```

The command only invokes the tested successor migration. It does not start the
console, load credentials, call a private endpoint, or create an order.

After source deployment and migration, restart through the secure OKX Demo
launcher. The operator must enter the three Demo credentials again because the
system never persists them. Do not attempt to recover process-only credentials
from the old process.

## Acceptance Checks

After secure restart, verify:

1. Console and workflow versions are `V13.27.9`.
2. Ten active successors use the Top100 policy and ten predecessors are
   archived with a manifest.
3. Public runtime is `warm=true` and `synchronized=true`.
4. The console shows the expected timeframes and a current confirmed close.
5. One timeframe uses one shared frozen snapshot across all due releases.
6. Duplicate symbols are arbitrated before order submission.
7. Stage latency metrics are present for a real confirmed-close cycle.
8. Expired and conditional late entries are explicitly counted.
9. Withdraw remains absent, Live remains gated, and raw credentials are not
   stored.

## Safety Boundary

- OKX Demo only for automatic strategy orders in this release.
- Immutable eligible Demo Release required.
- Existing portfolio and strategy risk gates required.
- No per-order confirmation in Demo after the runtime is explicitly armed.
- No raw API Key storage.
- No Withdraw integration.
- No automatic Live promotion.
- No expansion of Live permissions.
