# V13.27.1.61 Global Remediation

This release performs a bounded global remediation from frozen repository
heads. Findings are reclassified as `confirmed_present`, `already_fixed`,
`partially_fixed`, `obsolete`, or `not_evaluable`; only confirmed and partial
findings enter implementation.

The change introduces:

- a single-writer execution runtime lease;
- ordered technical-readiness, exact-approval, ARM, and lease gates;
- explicit emergency cancel, close-only, flatten, and kill semantics;
- point-in-time feature timestamps and null-preserving cost/PnL fields;
- versioned risk units, latency, and non-global exit-policy semantics;
- default read-only remote HTTP access with protected write requests;
- SQLite foreign-key, WAL, migration, backup, and integrity helpers;
- a bounded, credential-free Strategy Factory worker boundary;
- no-order deterministic and replay Shadow Parity;
- compact read-only Demo and Live UI V2 preview routes.

No production runtime cutover is performed in this release. The existing Demo
service remains untouched and may require process-only credentials to be ARM'd
again. Adaptive Learning remains blocked until real training, purged
walk-forward, Qlib, Demo decision-mode, drift, and rollback evidence exists.
Live and Withdraw remain disabled.

Final validation passed 655 Console tests and 426 Quant tests. Nineteen
non-empty production SQLite ledgers also passed online backup, restore,
schema-parity, and integrity checks on copies. Deterministic fixture and replay
Shadow Parity both reached 100%, but no production cutover was performed.
