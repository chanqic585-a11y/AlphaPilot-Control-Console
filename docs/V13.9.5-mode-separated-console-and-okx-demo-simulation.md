# V13.9.5 Mode-separated Console and OKX Demo Simulation

## Goal

V13.9.5 separates the AlphaPilot control console into clearer operating modes:

1. Local Sandbox and Local Simulation
2. OKX Demo Trading simulation
3. Future live trading, still locked

The goal is to make the console easier to operate while preserving strict safety
boundaries before any live trading work.

## UI Changes

- `Local Lab` combines local sandbox, local simulation equity, closed samples,
  strategy review status, and sandbox daily report actions.
- `OKX Demo` is a separate exchange-style page with connection status,
  credential status, read-only check, manual demo order ticket, emergency drill,
  blockers, and recent events.
- Older detailed observation/testnet panels are hidden behind Advanced Research
  mode so the main workflow is not cluttered.

## OKX Demo API Boundary

The OKX Demo page only reads credentials from environment variables:

- `ALPHAPILOT_OKX_DEMO_ENABLED`
- `ALPHAPILOT_OKX_DEMO_API_KEY`
- `ALPHAPILOT_OKX_DEMO_SECRET_KEY`
- `ALPHAPILOT_OKX_DEMO_PASSPHRASE`
- `ALPHAPILOT_OKX_DEMO_ORDER_ENABLED`
- `ALPHAPILOT_OKX_DEMO_CANCEL_ENABLED`

Raw API keys are not accepted through the UI and are not saved to browser state,
SQLite, or local console state.

OKX Demo private requests must include:

```text
x-simulated-trading: 1
```

Reference:

- [OKX API documentation](https://www.okx.com/docs-v5/en/)
- [OKX API FAQ](https://www.okx.com/en-us/help/api-faq)

## Demo Order Gate

Demo order submission requires all of these:

- `ALPHAPILOT_OKX_DEMO_ENABLED=1`
- `ALPHAPILOT_OKX_DEMO_ORDER_ENABLED=1`
- complete OKX Demo credentials
- user-entered `OKX_DEMO_ORDER_APPROVED`
- explicit OKX `sz`
- notional amount at or below 1000 USDT

If any condition is missing, the event is saved as blocked and no exchange order
request is sent.

## Emergency Drill Gate

Emergency stop can be saved as a local drill without sending an exchange cancel.

An OKX Demo cancel request requires all of these:

- `ALPHAPILOT_OKX_DEMO_ENABLED=1`
- `ALPHAPILOT_OKX_DEMO_CANCEL_ENABLED=1`
- complete OKX Demo credentials
- user-entered `OKX_DEMO_EMERGENCY_APPROVED`
- explicit `ordId`

## Safety Boundary

V13.9.5 does not enable live trading.

It does not enable:

- Withdraw API
- live API key entry
- raw API key storage
- real account trading
- automatic order creation
- automatic strategy execution
- live position management

The intended upgrade path remains:

```text
Local Sandbox -> Local Simulation -> OKX Demo -> manual review -> future live gate
```
