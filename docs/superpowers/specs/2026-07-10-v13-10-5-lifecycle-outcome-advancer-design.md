# V13.10.5 本地生命周期推进与闭合样本学习设计

## 产品目标

把 V13.10.4 中“只能看见但不会变化”的本地模拟持有记录推进为可审计的观察状态，并为未来闭合样本学习报告建立真实数据来源。整个流程只读取 OKX 公共行情和本地状态，不使用私有凭据，不创建任何交易所订单。

## 方案选择

### 方案 A：直接修改原始自动执行记录

实现最简单，但会破坏 V13.10.2 的不可变审计记录，后续无法判断某个字段何时改变，不采用。

### 方案 B：每次推进复制一条完整自动执行记录

保留历史，但会让生命周期记录数和策略样本数重复膨胀，统计容易把同一个观察当成多个候选，不采用。

### 方案 C：追加生命周期事件并投影最新状态

原始记录保持不变；每次基准建立、价格标记和闭合都追加事件。读取生命周期、复核队列和学习报告时，将最新事件投影到原记录。该方案可追溯、可回放、不会重复计数，作为 V13.10.5 实现方案。

## 数据来源

- 当前价：OKX public `GET /api/v5/market/ticker`。
- K 线：OKX public `GET /api/v5/market/candles`。
- 风险单位：最近确认 K 线计算的 ATR14。
- ATR 倍数：优先解析策略名称中的 `ATR1.0`、`ATR1.2` 等标记；无法解析时使用明确记录的 `1.0` 本地研究回退值。
- 原始候选、方向、周期和目标 R：本地 `autoExecutionRecords`。

不读取账户、持仓、订单、API Key 或交易所私有接口。

## 生命周期规则

### 第一次推进

对没有入场基准的活跃记录：

1. 获取公共 ticker 和 OHLCV。
2. 计算 ATR14。
3. 将当前 ticker 价格记录为“本地观察基准价”，不是历史成交价。
4. 计算 1R 风险距离、2R 目标价和 -1R 观察止损价。
5. 追加“基准已建立”事件，当前 R 固定为 0。
6. 第一次推进不允许立即判定盈利或亏损。

### 后续推进

对已有基准的活跃记录：

- 做多：`当前R = (当前价 - 基准价) / 1R价格距离`。
- 做空：`当前R = (基准价 - 当前价) / 1R价格距离`。
- 当前 R 达到目标 R：追加“达到 2R”闭合事件。
- 当前 R 小于等于 `-1R`：追加“触发 -1R”闭合事件。
- 超过观察窗口：追加“过期退出”闭合事件。
- 其他情况：追加“价格已更新”事件，保持本地模拟持有。
- 两次推进之间使用确认 K 线按时间顺序检查止盈止损触及；同一根 K 线双触发时无法确认先后，保守记录为止损并标注歧义。

观察窗口按周期设置：15m 为 96 根、1h 为 72 根、4h 为 42 根、1d 为 30 根。

## 数据模型

在 `console_state.json` 中新增兼容键：

```text
autoExecutionLifecycleEvents
```

每个事件至少包含：

```text
eventId
recordId
eventType
eventLabel
projection
marketSnapshot
createdAt
source
apiKeyUsed=false
ordersCreated=false
demoOrderCreated=false
liveTrading=false
```

`projection` 只保存需要覆盖到原记录的生命周期字段，不复制整条策略记录。

## 模块边界

- `public_exchange_registry.py`：新增单个 OKX 公共行情快照读取和 ATR14 计算所需的标准化 K 线输出。
- `state_store.py`：新增生命周期事件追加和读取函数。
- `auto_execution_lifecycle_advancer.py`：建立基准、推进状态、投影记录。
- `auto_execution_learning.py`：只统计已经闭合的投影记录，不在样本不足时训练模型。
- `auto_execution_lifecycle.py`、`auto_execution_review.py`、`auto_execution_engine.py`：改为读取投影后的记录。
- `http_app.py`：新增推进和学习接口。
- Web UI：新增“推进本地生命周期”按钮和闭合学习摘要。

## HTTP API

```text
POST /api/auto-execution-lifecycle/advance
GET  /api/auto-execution-learning
```

推进接口只触发公共行情读取和本地事件追加，不触发交易所 Dry-run、Demo 或实盘订单。

## 错误处理

- ticker 或 OHLCV 失败：该记录返回中文错误，不写入伪造价格。
- ATR14 不可计算：记录保持活跃，返回“ATR 数据不足”。
- 风险距离为零或无效：记录保持活跃，不计算 R。
- 已闭合记录：不重复推进。
- 单个币种失败：不影响其他活跃记录。
- 没有活跃记录：返回空推进结果，不报错。

## 学习报告边界

- 闭合样本少于 30：只生成描述性统计，状态为“闭合样本不足”。
- 闭合样本达到 30：允许初步比较，但不自动晋级。
- 闭合样本达到 100：允许严肃复核。
- 闭合样本达到 300：允许稳定性判断。
- V13.10.5 不训练机器学习模型，不自动修改策略，不输出实盘建议。

## 验证

1. 原始 `autoExecutionRecords` 数量和内容保持不变。
2. 首次推进只建立基准，活跃记录当前 R 为 0。
3. 再次推进可产生价格标记事件，但不得重复初始化。
4. 人工构造的单元样本可以覆盖 2R、-1R 和过期三种闭合状态。
5. 生命周期、复核和学习接口读取同一份投影结果。
6. 页面显示中文状态、价格来源、ATR 风险模型和样本门槛。
7. Python 编译、JavaScript 检查、smoke、接口和浏览器检查通过。
8. 安全扫描确认没有私有 API、订单创建或自动交易能力。
